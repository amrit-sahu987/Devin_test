## Quick Start Guide (Local Setup)

To test this orchestrator locally, you do not need to install Python or ngrok on your host machine. Everything is containerized.

### Prerequisites
1. [Docker Desktop](https://www.docker.com/products/docker-desktop) installed and running.
2. A free [ngrok account](https://dashboard.ngrok.com/signup) to generate an Auth Token.
3. A [Devin](https://devin.ai) API Key.
4. A GitHub Personal Access Token (PAT) with `repo` scope (so Devin can push branches).

### Step 1: Fork and Clone
Because GitHub Actions are tied to specific repositories, you must run this on your own fork.
```bash
git clone [https://github.com/YOUR_USERNAME/YOUR_FORKED_REPO.git](https://github.com/YOUR_USERNAME/YOUR_FORKED_REPO.git)
cd YOUR_FORKED_REPO

### Step 2: Configure Environment Variables
Copy the template environment file and add your credentials.

cp .env.example .env
Open .env and fill in your keys:

DEVIN_API_KEY: Your key from the Devin settings.

GITHUB_TOKEN: Your GitHub PAT.

NGROK_AUTHTOKEN: Your free token from the ngrok dashboard.

### Step 3: Start the Infrastructure
Run the application using Docker Compose. This spins up both the FastAPI orchestrator and the ngrok tunnel simultaneously.

docker-compose up --build
Once running, open your browser and navigate to the local ngrok dashboard: http://localhost:4040.
Copy the public URL generated by ngrok (e.g., https://xyz.ngrok-free.app).

### Step 4: Configure GitHub Actions
Link your repository to your local orchestrator:

Go to your forked repository on GitHub.

Navigate to Settings > Secrets and variables > Actions.

Create a new Repository Secret named ORCHESTRATOR_URL.

Paste the ngrok URL you copied in Step 3 (ensure there is no trailing slash).

### CI/CD Workflow Configuration
Ensure your GitHub Actions workflow file (e.g., .github/workflows/security-scan.yml) is configured to ignore the branches that Devin creates. This prevents an infinite bot loop where Devin's own commits trigger new security scans.

on:
  push:
    # Run on all branches EXCEPT the one the agent creates
    branches-ignore:
      - 'fix/ci-security-patch'
  workflow_dispatch:

### Testing the Pipeline
To see the autonomous agent in action:

Create a new branch.

Downgrade a dependency in your project (e.g., requests==2.28.1 in requirements.txt) to introduce a known vulnerability.

Commit and push the branch to trigger the CI/CD pipeline.

The Automated Lifecycle:

GitHub Actions will fail the security scan.

The Action will fire a webhook payload containing the failure details to your ngrok URL.

The local Docker container will catch the payload and initialize a new Devin session.

Devin will autonomously clone the repo, fix the dependency, and open a Pull Request.
